# Project Plan for Notion-like Editor with Git Integration

This document outlines the comprehensive plan for developing a Kanban-style task tracking system backed by GitHub issues. The project leverages Docable's GitHub app for seamless integration.

## Task Overview

Task is to create a Kanban-style task management system that synchronises with GitHub issues.

## Key Features

- Kanban-style task tracking
- Integration with GitHub issues
- Utilisation of Docable's GitHub app

## Workflow

Github issue should be our single source of truth. Our firebase should always be updated according to Github and Docable client should reflect firebase. There will be the following two sequence of data flow.

1. Docable -&gt; Firebase Function -&gt; Firestore && Github
2. Github -&gt; WebHook -&gt; update Firestore -&gt; Docable

### Firestore Collection

Collection : `projects/{projectId}/issues`

Github User Id : `users/{userId}/integrations/{githubinfo_id}`

### Callable and WebHooks

**Docable -&gt; Github** - in this case docable client will call the appropriate firebase functions. There will following functions -

- `CreateIssue` - This creates a Github issue and if successful creates a issue object in firestore. The below link documents the REST API for creating an issue.\
  <https://docs.github.com/en/rest/issues/issues?apiVersion=2022-11-28#create-an-issue>
- `UpdateIssue` - This updates any required field in Github. The below link documents the REST API to update an issue. <https://docs.github.com/en/rest/issues/issues?apiVersion=2022-11-28#update-an-issue>
- `AssignIssue` - This allows to assign this issue to particular user. This will assign the issue to the given list of users using their `github_id` .

**Github -&gt; Docable -** If in case user directly interacts with a issue in Github we will need to update the firestore document to reflect the change. For this we will use WebHooks via the Docable Github app.

For details related to the Webhook - <https://docs.github.com/en/webhooks/webhook-events-and-payloads?actionType=assigned#issues>
